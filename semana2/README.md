## Resumo da Aula: Classes, Objetos e Boas Práticas em C#

Esta aula abordou os conceitos fundamentais de programação orientada a objetos em C#, com foco em classes, objetos, organização do código e boas práticas de semântica. A seguir, um resumo detalhado dos principais tópicos estudados:

### 1. Classes e Objetos
- **Classe**: Estrutura que define um tipo personalizado, agrupando dados (fields) e comportamentos (métodos).
- **Objeto**: Instância de uma classe, criado para representar entidades concretas no programa.
- Por padrão, classes são privadas, reforçando o encapsulamento e a segurança dos dados.

### 2. Estrutura de Classes
- **Fields**: Variáveis declaradas dentro de uma classe, responsáveis por armazenar o estado dos objetos.
- **Métodos**: Funções definidas dentro da classe, que representam comportamentos e operações sobre os dados.
- **Class Members**: Termo que engloba fields, métodos, propriedades e eventos de uma classe.

### 3. Organização do Código
- Cada aplicação deve conter apenas um método `Main`, responsável pelo ponto de entrada do programa.
- É possível definir múltiplas classes em um mesmo arquivo, facilitando a organização e reutilização do código.
- A separação clara entre definição de classe e instanciamento de objetos contribui para a legibilidade e manutenção do código.

### 4. Exemplos Práticos
- Criação de classes com fields públicos e privados.
- Instanciação de objetos e acesso aos fields e métodos.
- Modificação de fields após a criação do objeto, útil para cenários com múltiplas instâncias.

### 5. Boas Práticas
- Utilizar nomes claros e descritivos para classes, fields e métodos, facilitando o entendimento do código.
- Manter o encapsulamento, restringindo o acesso direto aos dados sempre que possível.
- Organizar o código em arquivos e classes de acordo com a responsabilidade de cada componente.
- Evitar duplicidade do método `Main` na aplicação.

### 6. Semântica
- A semântica correta na definição de classes e membros garante que o código seja intuitivo e mantenível.
- O uso adequado de modificadores de acesso (public, private) reforça a segurança e o encapsulamento.

---
Este conteúdo serve como base para o desenvolvimento de aplicações orientadas a objetos em C#, promovendo organização, clareza e boas práticas de programação.
